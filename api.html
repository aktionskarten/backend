GET /api/maps/(string: map_id).(string: file_type)/status¶

Retrieve status of a render job by it map_id, verison and file_type

Parameters

  • map_id – map id

  • version – version

  • file_type – filetype

Status Codes

  • 200 OK – informations about current job

  • 400 Bad Request – invalid file type

  • 404 Not Found – map and or version of map not found

GET /api/maps/(string: map_id)/render/(string: file_type)¶

Renders a map

Example request:

$ curl -i -X POST -H "Content-Type: application/json" -d @tests/data/test_map.json http://localhost:5000/render/png

Example response:

HTTP/1.1 202 Accepted
Content-Type: application/json

{
  "file_type": "png",
  "job_id": "fce3b682-feaa-48b7-b945-ad243ce62df4",
  "map_id": "test123",
  "status": "queued",
  "version": "c8bbde29fa1cfd1109fae1835fcc1ea1f92f4e31292c1b3d338c782f18333605"
}
Parameters

  • file_type – file type of rendered map. Either pdf, svg or png:<size> with size small, medium or large

GET /maps/(string: map_id).(string: file_type)/(string: version)¶
GET /maps/(string: map_id).(string: file_type)¶

Download a map identified by map_id, file_type and optional version.

Only already rendered maps can be downloaded. Each file format has to be rendered separately. If a map is not found you will get an 404 error statuscode.

Parameters

  • map_id – id of map

  • file_type – svg, pdf or png[:small|medium|large]

Status Codes

  • 200 OK – sends content of map

  • 404 Not Found – map, version or file_type not found
